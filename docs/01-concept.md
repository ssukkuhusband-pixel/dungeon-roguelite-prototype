# 게임 컨셉 문서

## 한 줄 컨셉
영웅을 수집하고 편성하여, 횡스크롤 턴제 자동전투로 던전을 공략하며 로그라이트 스킬 빌드를 완성해나가는 수집형 RPG

## 장르
- 메인 장르: 수집형 RPG (가챠 RPG)
- 서브 장르: 횡스크롤 턴제 자동전투 + 로그라이트

## 핵심 재미 요소
- **로그라이트 스킬 선택**: 전투 승리마다 3개 중 1개 스킬을 골라 빌드를 쌓아가는 궁수의 전설식 성장
- **편성의 묘미**: 4명의 영웅 조합으로 시너지를 만드는 수집형 RPG의 재미
- **횡스크롤 자동전투**: 다키스트 던전풍 4:4 횡스크롤에서 펼쳐지는 자동전투의 시각적 쾌감
- **던전 탐험의 긴장감**: 10스테이지 구성 + 이벤트 스테이지로 매 런마다 다른 경험

## 레퍼런스 게임
| 게임명 | 참고 포인트 |
|--------|------------|
| 궁수의 전설 (Archero) | 로그라이트 스킬 선택 시스템, 스테이지 진행 구조 |
| 다키스트 던전 (Darkest Dungeon) | 횡스크롤 4인 파티 전투 비주얼, 어둡고 긴장감 있는 던전 탐험 |
| AFK 아레나 | 수집형 영웅 시스템, 자동전투, 캠프(모닥불) 느낌의 메인 화면 |

## 타겟 유저
- 수집형 RPG를 즐기는 모바일/웹 게임 유저
- 짧은 세션 (한 런 5~10분)으로 가볍게 즐기고 싶은 유저
- 로그라이트의 매 판 다른 빌드 경험을 좋아하는 유저

## 톤 & 무드
- 다크 판타지 + 약간의 스타일리시함
- 다키스트 던전의 무거운 분위기를 기반으로, 수집형 RPG답게 캐릭터는 매력적으로
- 메인 화면(모닥불 캠프)은 따뜻하고 아늑한 분위기 / 던전 안은 어둡고 긴장감 있는 분위기

## 프로토타입 스코프
### 핵심 경험 (반드시 구현)
"4명의 영웅을 편성하고 던전에 입장하여, 횡스크롤 자동전투로 적과 싸우고, 승리 후 3개 스킬 중 1개를 골라 빌드를 쌓으며 10스테이지를 클리어하는 한 번의 런"

### 포함 범위
- 메인 화면 (모닥불 캠프, 챕터 선택, 입장)
- 횡스크롤 4:4 턴제 자동전투
- 로그라이트 스킬 선택 (전투 승리 후 3택)
- 10스테이지 1챕터 진행
- 이벤트 스테이지 1종 (쉼터: HP 회복)
- 영웅 목록 화면 (보유 영웅 확인)
- 영웅 뽑기 화면 (기본 가챠 1종)
- 영웅 4~6종 (프로토타입용 기본 캐릭터)

### 명시적 제외
- 다중 챕터 (챕터 1만 구현, 좌우 화살표 UI만 존재)
- 장비 시스템
- PvP / 소셜 기능
- 튜토리얼
- 설정 / 옵션
- 과도한 영웅 볼륨 (6종 이내)
- 실제 결제 연동

## 코어 디자이너 브리핑
횡스크롤 4:4 턴제 자동전투가 핵심이다. 다키스트 던전처럼 왼쪽에 아군 4명, 오른쪽에 적 4명이 배치되고, 각자 턴이 돌아오면 자동으로 스킬을 사용한다. 전투 승리 후 궁수의 전설식 스킬 3택이 핵심 재미다. 10스테이지를 하나의 런으로 구성하고, 중간에 이벤트 스테이지가 끼어들 수 있다. 속도감 있는 자동전투와 스킬 선택의 전략적 긴장감 사이 밸런스가 중요하다.

## 메타 디자이너 브리핑
수집형 RPG의 기본 구조를 따른다. 영웅 수집(가챠), 영웅 성장은 프로토타입에서는 최소한으로 구현한다. 핵심은 "던전 클리어 보상 → 재화 획득 → 영웅 뽑기 → 더 강한 편성 → 더 높은 챕터 도전"의 루프가 느껴지는 것이다. 프로토타입에서는 1챕터만 있으므로, 보상으로 뽑기 재화를 주고 뽑기에서 새 영웅을 얻는 것까지가 범위다.

## 아트 디렉터 브리핑
다크 판타지 톤이 기본이지만, 수집형 RPG답게 캐릭터는 매력적이어야 한다. 메인 화면의 모닥불 캠프는 따뜻한 조명감, 던전 내부는 어둡고 긴장감 있는 분위기. 횡스크롤 전투 화면이 가장 중요한 비주얼이므로, 캐릭터 일러스트와 전투 배경에 집중한다. 픽셀아트보다는 일러스트/세미 리얼 방향이 수집형 RPG의 캐릭터 매력을 살리기 좋다.
